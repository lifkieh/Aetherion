# FINAL PROJECT STRUCTURE

```text
TIER S
Foundation

TIER A
Systems

TIER B
Living World

TIER C
Content

TIER D
Production
```

Dengan struktur ini, seluruh proyek Aetherion dapat dikelola hanya dengan 5 lapisan besar tanpa harus memecah roadmap awal menjadi ratusan dokumen sejak awal pengembangan.
SKILL DESIGN

Skill Tree bersifat:

Horizontal First
Vertical Second

Artinya:

lebih banyak pilihan

daripada angka.

COMBAT DESIGN

Combat harus:

mudah dipahami
sulit dikuasai
mendukung kreativitas

Combat bukan:

Spam skill terbesar.
CREATURE DESIGN

Semua monster adalah:

Makhluk hidup.

Bukan resource berjalan.

Monster harus memiliki:

habitat
perilaku
kebutuhan
hubungan ekologis
KINGDOM DESIGN

Kerajaan bukan dekorasi.

Kerajaan adalah:

aktor politik
aktor ekonomi
aktor militer
aktor budaya
NPC DESIGN

NPC bukan quest giver.

NPC adalah:

Penduduk dunia.
ECONOMY DESIGN

Ekonomi harus:

menghasilkan sesuatu
mengonsumsi sesuatu
memindahkan sesuatu

Tidak boleh ada ekonomi palsu.

EXPLORATION DESIGN

Exploration harus memberi:

pengetahuan
peluang
cerita
kejutan

Bukan hanya loot.

MULTIPLAYER DESIGN

Pemain harus:

bisa bekerja sama
bisa bersaing
bisa hidup berdampingan

Tanpa harus saling membunuh.
