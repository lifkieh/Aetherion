DESIGN PHILOSOPHY

Salah satu alasan banyak RPG terasa mati adalah:

Dunia hanya berisi:

Quest Giver
Merchant
Blacksmith
Guard

Padahal dunia nyata jauh lebih aneh.

Lebih berantakan.

Lebih tidak masuk akal.

Dan justru itu yang membuatnya hidup.

Maka Aetherion harus memiliki:

The People Between History.

Orang-orang yang tidak akan menyelamatkan dunia.

Tetapi membuat dunia terasa nyata.

THE SHADOW MARKET
FACTION #001

Pasar gelap terbesar dunia.

Bukan organisasi.

Lebih seperti ekosistem.

Mereka menjual:

artefak ilegal
monster langka
informasi
identitas palsu
peta terlarang

Tidak ada pemimpin tunggal.

Karena ketika satu pemimpin mati.

Pasar tetap hidup.

Sebagian dianggap gila.

Baik

THE RULE OF WEIRDNESS

Minimal 30% dunia harus terasa aneh.

Karena rasa penasaran adalah bahan bakar eksplorasi.

Contoh:

Pemain menemukan pria tua.

Ia meminta:

Tolong hitung jumlah batu di sungai ini.

Quest terlihat bodoh.

100 jam kemudian.

Pemain sadar pria itu adalah salah satu Companion legendaris.

THE RULE OF IRRELEVANT PEOPLE

Tidak semua NPC penting.

Dan itu penting.

Karena jika semua NPC penting.

Maka tidak ada NPC yang terasa penting.

Harus ada:

petani biasa
penjudi gagal
pemabuk
penipu kecil
pencari kerja
pedagang keliling

Agar dunia terasa hidup.

THE FINAL LAW

Tidak semua orang sedang menunggu pemain.

Sebagian besar sedang menjalani hidup mereka sendiri.

Dan itulah yang membuat dunia terasa nyata.

# DESIGN REASONING BESAR

Ini mengunci sesuatu yang sebelumnya sering Anda sebut:

Mereka bukan quest.

Mereka bukan konten sampingan.

Mereka adalah "isi dunia".

Karena pemain harus bisa berjalan 10 menit dan menemukan sesuatu yang tidak direncanakan.

TEMUAN DIREKTUR #030

Sekarang saya melihat fondasi 50 jam pertama mulai lengkap.

Tetapi ada satu sistem yang akan menentukan seluruh gameplay:

CLASS SYSTEM

Karena dari visi awal Anda:

Tamer + Warrior
Tamer + Necromancer
Tamer + Kingdom Builder
Tamer + Farmer
Tamer + Lumberjack

Artinya kita tidak sedang membuat class tradisional MMORPG.

Kita sedang membuat:

Identity System

bukan sekadar class system.

DECISION LOCK #031
CLASS IS A STARTING IDENTITY
NOT A PERMANENT LIMITATION
PROBLEM OF MOST RPG

Biasanya:

Warrior
=
Tidak bisa sihir

Mage
=
Tidak bisa tank

Archer
=
Tidak bisa summon

Aetherion tidak cocok seperti itu.

Karena visi awal Anda adalah:

Aku punya build ku sendiri.

Bukan:

Aku memainkan build yang disediakan developer.
THE CORE RULE
ALL SKILLS CAN BE LEARNED

Semua skill tree bisa dipelajari.

Contoh:

Warrior dapat mempelajari:

necromancy
taming
crafting
farming
rune magic

Mage dapat mempelajari:

sword mastery
beast handling
blacksmithing
leadership

Tamer dapat mempelajari:

shield techniques
curse magic
kingdom administration
THEN WHAT IS A CLASS?

Class hanya menentukan:

Starting Identity

Contoh:

Warrior

Mulai dengan:

pedang
armor
combat skills
Tamer

Mulai dengan:

monster bond
taming skills
pet command
Necromancer

Mulai dengan:

soul manipulation
undead control

Dan seterusnya.

IMPORTANT
NO SKILL TREE LOCK

Tidak ada:

Class Requirement

untuk sebagian besar skill.

Karena dunia tidak peduli label.

Yang penting adalah:

Pengetahuan
+
Latihan
+
Syarat tertentu
THE ONLY EXCEPTION
CLASS ULTIMATE

Ini bagian yang Anda tentukan.

Setiap class memiliki:

Class Core

Dan:

Class Ultimate

Yang tidak dapat dipelajari class lain.

EXAMPLE
Tamer Ultimate

Misalnya:

Primal Union

Menyatu dengan monster partner.

Transformasi penuh.

Tidak bisa dipelajari Warrior.

Tidak bisa dipelajari Mage.

Tidak bisa dipelajari Necromancer.

EXAMPLE
Warrior Ultimate

Misalnya:

Worldbreaker Stance

Teknik tempur tertinggi Warrior.

Tidak dapat dipelajari class lain.

EXAMPLE
Necromancer Ultimate

Misalnya:

Throne of the Dead

Kontrol massal pasukan undead.

Tidak dapat dipelajari class lain.
